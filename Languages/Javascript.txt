

# **JavaScript**

### **1. Language Basics**

* **Type System:** Dynamically typed, weakly typed. Types are inferred at runtime; implicit type coercion can occur.
* **Paradigm:** Multi-paradigm â€” supports procedural, object-oriented, and functional programming.
* **Syntax Overview:**

  * Variables: `let x = 10;`, `const y = 5;`, `var z = 3;`
  * Functions:

    ```javascript
    function add(a, b) {
        return a + b;
    }
    // Arrow function
    const multiply = (a, b) => a * b;
    ```
  * Classes:

    ```javascript
    class Node {
        constructor(val) {
            this.val = val;
            this.next = null;
        }
    }
    ```
  * Loops: `for (let i = 0; i < n; i++)`, `while (condition)`, `for...of`
  * Conditionals: `if`, `else if`, `else`, ternary `condition ? expr1 : expr2`

---

### **2. Data Structures**

* **Built-in:** Arrays, Objects, `Set`, `Map`, `WeakSet`, `WeakMap`.
* **Custom Structures:** Classes can be used for linked lists, trees, graphs; objects often used as hash maps.
* **Immutability:** Primitive types (numbers, strings, booleans) are immutable; arrays and objects are mutable.
* **Syntax Differences:**

  * Array: `let arr = [1,2,3];`
  * Object: `let obj = {a:1, b:2};`
  * Set: `let s = new Set([1,2,3]);`
  * Map: `let m = new Map(); m.set('a', 1);`

---

### **3. Algorithms & Language Support**

* **Standard Library:** `Math`, `Array` methods, `Set`/`Map` utilities, `JSON`.
* **Recursion:** Fully supported; no tail recursion optimization in most engines.
* **Iteration Constructs:** `for`, `while`, `for...of`, `for...in`, `map`, `filter`, `reduce`.
* **Built-in Helpers:**

  * Sorting: `arr.sort((a,b)=>a-b)`
  * Searching: `arr.indexOf(x)`, `arr.includes(x)`
  * Math: `Math.max(a,b)`, `Math.min(a,b)`, `Math.pow(a,b)`

---

### **4. Performance & Complexity**

* **Time Complexity Notes:** Slower than C++ and Java for intensive computation; similar to Python in practice.
* **Memory Usage:** Moderate; garbage collected automatically.
* **Concurrency Support:** Single-threaded; async tasks via Promises and `async/await`, event loop handles concurrency.
* **Compiler/Interpreter Optimizations:** Runs on V8 (Chrome, Node.js) or other JS engines with JIT compilation.

---

### **5. Syntax Specifics for LeetCode**

* **Function Signature:** Usually a standard function or method inside `class Solution`.
* **Class vs Static Functions:** Can define methods inside class or standalone functions.
* **Input/Output Handling:** Arrays and objects are common; trees and linked lists represented with objects.
* **Edge Case Handling:** `null` and `undefined` require explicit checks; automatic type coercion can cause unexpected results.

---

### **6. Use Cases / Problem Type Suitability**

* **Commonly Solved Problems:** Arrays, strings, hash maps, functional-style operations, recursion.
* **Strengths:** Concise syntax, first-class functions, flexible dynamic types.
* **Weaknesses:** Slower execution for heavy computations, potential pitfalls with type coercion, less strict error checking.

---

### **7. Language Idioms & Best Practices**

* Use `const` for constants and `let` for variables that change.
* Functional array operations:

  ```javascript
  arr.map(x => x*2).filter(x => x>5);
  ```
* Object destructuring:

  ```javascript
  const {a, b} = obj;
  ```
* Optional chaining to handle nulls: `obj?.property?.subproperty`

---

### **8. Error Handling & Debugging**

* **Exception Handling:**

  ```javascript
  try {
      // code
  } catch (e) {
      console.error(e);
  }
  ```
* **Common Pitfalls:** Type coercion issues, `undefined` vs `null`, off-by-one errors.
* **Debugging Support:** `console.log`, `debugger` statement, Node.js/Chrome DevTools.
